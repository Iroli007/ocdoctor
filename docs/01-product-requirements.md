# 产品需求文档

## 用户痛点

用户当前痛点不是"没有资料"，而是：

1. 重点都在书上和讲义上，难以快速转成可复习材料
2. 手动整理卡片很费时间
3. 会背，但不容易比较相似方剂
4. 学完之后很难进入高频复习状态

## MVP 功能定义

### 必做功能

1. **用户创建学习项目** - 创建学习集合（如"方剂学-解表剂"）
2. **用户输入资料**
   - 文字粘贴
   - 图片上传
   - 图片 OCR 导入
3. **系统抽取并结构化内容** - AI 提取关键字段
4. **系统生成学习卡片** - 生成方剂知识卡片
5. **系统生成比较题** - 生成相似方剂比较题
6. **用户开始复习** - 抽认卡、选择题模式
7. **记录答题结果与错题** - 跟踪学习进度

### 不做功能

- 用户社交
- 大规模知识库检索
- 多角色体系
- 支付系统
- 原生 iOS/Android
- 复杂病例模拟器
- 大模型连续对话工作台

## 成功标准

MVP 成功标准：
1. 用户能否在 3 分钟内完成一次"导入内容 → 得到卡片"
2. 用户是否愿意在第二天回来继续复习
3. 用户是否觉得"比手工整理省时间"
